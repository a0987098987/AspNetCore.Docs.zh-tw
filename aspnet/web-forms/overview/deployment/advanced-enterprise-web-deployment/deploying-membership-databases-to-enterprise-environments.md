## <a name="what-are-the-issues-when-you-deploy-a-membership-database"></a><span data-ttu-id="6f8d7-111">當您部署成員資格資料庫時，為何的問題？</span><span class="sxs-lookup"><span data-stu-id="6f8d7-111">What Are the Issues When You Deploy a Membership Database?</span></span>

<span data-ttu-id="6f8d7-112">在大部分情況下，您設計部署策略的資料庫，您需要考慮的第一件事時您想要部署哪些的資料。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-112">In most cases, when you devise a deployment strategy for a database, the first thing you need to consider is what data you want to deploy.</span></span> <span data-ttu-id="6f8d7-113">在開發或測試環境中，您可能想要部署使用者帳戶資料，以促進快速和輕鬆測試。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-113">In a development or test environment, you might want to deploy user account data to facilitate quick and easy testing.</span></span> <span data-ttu-id="6f8d7-114">在預備或生產環境中，為幾乎不太可能您會想要部署使用者帳戶資料。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-114">In a staging or production environment, it's very unlikely that you'd want to deploy user account data.</span></span>

<span data-ttu-id="6f8d7-115">不幸的是，ASP.NET 成員資格資料庫導入做出此決策更加複雜的一些特定挑戰：</span><span class="sxs-lookup"><span data-stu-id="6f8d7-115">Unfortunately, ASP.NET membership databases introduce some specific challenges that make this decision a lot more complex:</span></span>

- <span data-ttu-id="6f8d7-116">僅限結構描述的部署將會保留成員資格資料庫處於非運作狀態。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-116">A schema-only deployment will leave the membership database in a non-operational state.</span></span> <span data-ttu-id="6f8d7-117">這是因為成員資格資料庫所包含的某些組態資料 (在**aspnet\_SchemaVersions**資料表) 的資料庫所需才能運作。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-117">This is because the membership database includes some configuration data (in the **aspnet\_SchemaVersions** table) that the database requires in order to function.</span></span> <span data-ttu-id="6f8d7-118">因此，如果您執行僅限結構描述部署的成員資格資料庫，以排除使用者帳戶資料，您必須在執行部署後指令碼中加入必要的設定資料。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-118">As such, if you perform a schema-only deployment of your membership database in order to exclude user account data, you'll need to run a post-deployment script to add the essential configuration data.</span></span>
- <span data-ttu-id="6f8d7-119">根據成員資格資料庫的設定方式，成員資格提供者可能會使用電腦金鑰加密的密碼，並將它們儲存在資料庫中。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-119">Depending on how your membership database is configured, the membership provider may use the machine key to encrypt passwords and store them in the database.</span></span> <span data-ttu-id="6f8d7-120">在此情況下，您部署與資料庫的任何使用者帳戶資料將會導致在目的地伺服器上使用。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-120">In this case, any user account data you deploy with the database will become unusable on the destination server.</span></span> <span data-ttu-id="6f8d7-121">基於這個理由，部署使用者帳戶的資料不支援的案例。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-121">For this reason, deploying user account data is not a supported scenario.</span></span>

## <a name="choosing-a-membership-database-strategy"></a><span data-ttu-id="6f8d7-122">選擇成員資格資料庫策略</span><span class="sxs-lookup"><span data-stu-id="6f8d7-122">Choosing a Membership Database Strategy</span></span>

<span data-ttu-id="6f8d7-123">當您選擇如何佈建企業伺服器環境中的成員資格資料庫，請使用下列方針：</span><span class="sxs-lookup"><span data-stu-id="6f8d7-123">Use these guidelines when you choose how to provision a membership database in an enterprise server environment:</span></span>

- <span data-ttu-id="6f8d7-124">可能的情況下，不會部署成員資格資料庫。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-124">Wherever possible, do not deploy membership databases.</span></span> <span data-ttu-id="6f8d7-125">相反地，手動在目標資料庫伺服器上建立成員資格資料庫。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-125">Instead, create the membership database manually on the target database server.</span></span> <span data-ttu-id="6f8d7-126">如果您未自訂成員資格資料庫結構描述，您可以只建立一個新的 situ 在目的地使用[ASP.NET SQL Server 註冊工具 (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx)。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-126">If you haven't customized your membership database schema, you can simply create a new one in situ at the destination using the [ASP.NET SQL Server Registration Tool (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span></span>
- <span data-ttu-id="6f8d7-127">如果您不有任何選項，但若要部署的成員資格資料庫 & #x 2014; 例如，如果您已擬定廣泛修改資料庫結構描述 & #x 2014; 您應該執行的成員資格資料庫，以排除使用者帳戶資料，僅限結構描述部署和然後執行部署後指令碼中加入任何必要的組態資料。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-127">If you have no option but to deploy a membership database&#x2014;for example, if you've made extensive modifications to the database schema&#x2014;you should perform a schema-only deployment of the membership database, to exclude user account data, and then run a post-deployment script to add any required configuration data.</span></span> <span data-ttu-id="6f8d7-128">您可以在這些方法中找到廣泛指導[How to： 部署 ASP.NET 成員資格資料庫但不包括使用者帳戶](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx)。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-128">You can find broad guidance on these approaches in [How to: Deploy the ASP.NET Membership Database Without Including User Accounts](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span></span>

<span data-ttu-id="6f8d7-129">請務必記住*的成員資格資料庫結構描述很可能是相當靜態*。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-129">It's important to remember that *the schema of your membership database is likely to be fairly static*.</span></span> <span data-ttu-id="6f8d7-130">即使您已自訂成員資格資料庫，不太確定您要更新的定期 & #x 2014年的結構描述; 不會變更與 web 應用程式或資料庫專案中的程式碼的頻率相同。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-130">Even if you've customized the membership database, it's unlikely that you'll need to update the schema on a regular basis&#x2014;it's not going to change with the same frequency as the code in a web application or a database project.</span></span> <span data-ttu-id="6f8d7-131">因此，您應該不需要包含任何自動或逐步部署程序中的成員資格資料庫。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-131">As such, you shouldn't need to include the membership database in any automated or single-step deployment processes.</span></span>

## <a name="using-vsdbcmd-to-update-a-membership-database-schema"></a><span data-ttu-id="6f8d7-132">使用 VSDBCMD 更新成員資格資料庫結構描述</span><span class="sxs-lookup"><span data-stu-id="6f8d7-132">Using VSDBCMD to Update a Membership Database Schema</span></span>

<span data-ttu-id="6f8d7-133">如果您在第一個部署後修改的成員資格資料庫結構，您可能不想要重新部署資料庫使用網際網路資訊服務 (IIS) Web Deployment Tool (Web Deploy)。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-133">If you modify the structure of your membership database after your first deployment, you may not want to use the Internet Information Services (IIS) Web Deployment Tool (Web Deploy) to redeploy the database.</span></span> <span data-ttu-id="6f8d7-134">Web 部署的資料庫部署功能不包含可讓目的地資料庫 & #x 2014年差異的更新功能; 相反地，Web Deploy 必須卸除並重新建立資料庫。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-134">The database deployment functionality in Web Deploy doesn't include the capability to make differential updates to a destination database&#x2014;instead, Web Deploy must drop and re-create the database.</span></span> <span data-ttu-id="6f8d7-135">這表示，就會失去任何現有的使用者帳戶資料，也就是在預備或生產環境中通常不需要。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-135">This means that you lose any existing user account data, which is typically undesirable in staging or production environments.</span></span>

<span data-ttu-id="6f8d7-136">替代方案是使用 VSDBCMD 公用程式來更新目的地資料庫的結構描述。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-136">The alternative is to use the VSDBCMD utility to update the schema of your destination database.</span></span> <span data-ttu-id="6f8d7-137">VSDBCMD 包括兩個重要的功能。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-137">VSDBCMD includes two important capabilities.</span></span> <span data-ttu-id="6f8d7-138">首先，它可以將現有的資料庫結構描述匯.dbschema 檔案。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-138">First, it can import the schema of an existing database into a .dbschema file.</span></span> <span data-ttu-id="6f8d7-139">第二，它可以部署.dbschema 檔案至現有的資料庫差異的更新，這表示它只會使目標資料庫的最新狀態所需的變更，而且不會遺失任何資料。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-139">Second, it can deploy a .dbschema file to an existing database as a differential update, which means that it only makes the changes required to bring the target database up to date and you don't lose any data.</span></span>

<span data-ttu-id="6f8d7-140">您可以使用下列高階步驟來更新成員資格資料庫結構描述：</span><span class="sxs-lookup"><span data-stu-id="6f8d7-140">You can use these high-level steps to update a membership database schema:</span></span>

2. <span data-ttu-id="6f8d7-143">使用 VSDBCMD**部署**部署.dbschema 檔案到目的地的成員資格資料庫中的動作。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-143">Use the VSDBCMD **Deploy** action to deploy the .dbschema file to your destination membership database.</span></span> <span data-ttu-id="6f8d7-144">此程序所述[VSDBCMD 的命令列參考。EXE （部署和結構描述匯入）](https://msdn.microsoft.com/library/dd193283.aspx)。</span><span class="sxs-lookup"><span data-stu-id="6f8d7-144">This procedure is described in [Command-Line Reference for VSDBCMD.EXE (Deployment and Schema Import)](https://msdn.microsoft.com/library/dd193283.aspx).</span></span>
