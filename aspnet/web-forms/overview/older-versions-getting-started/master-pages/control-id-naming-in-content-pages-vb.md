## <a name="introduction"></a><span data-ttu-id="b9cde-109">簡介</span><span class="sxs-lookup"><span data-stu-id="b9cde-109">Introduction</span></span>

<span data-ttu-id="b9cde-110">所有的 ASP.NET 伺服器控制項包括`ID`屬性，可唯一識別控制項，並會用該控制項以程式設計方式存取程式碼後置類別中的方法。</span><span class="sxs-lookup"><span data-stu-id="b9cde-110">All ASP.NET server controls include an `ID` property that uniquely identifies the control and is the means by which the control is programmatically accessed in the code-behind class.</span></span> <span data-ttu-id="b9cde-111">同樣地，在 HTML 文件中的項目可能包括`id`屬性可唯一識別項目; these `id` values are often used in client-side script to programmatically reference a particular HTML element.</span></span> <span data-ttu-id="b9cde-112">如此一來，您可能會假設，當 ASP.NET 伺服器控制項轉譯為 HTML，其`ID`值作為`id`所呈現的 HTML 項目的值。</span><span class="sxs-lookup"><span data-stu-id="b9cde-112">Given this, you may assume that when an ASP.NET server control is rendered into HTML, its `ID` value is used as the `id` value of the rendered HTML element.</span></span> <span data-ttu-id="b9cde-113">這不一定如此因為在某些情況下使用單一來控制單一`ID`值可能會多次出現在呈現的標記。</span><span class="sxs-lookup"><span data-stu-id="b9cde-113">This is not necessarily the case because in certain circumstances a single control with a single `ID` value may appear multiple times in the rendered markup.</span></span> <span data-ttu-id="b9cde-114">請考慮 GridView 控制項，其中包含與使用標籤 Web 控制項為 TemplateField`ID`的值`ProductName`。</span><span class="sxs-lookup"><span data-stu-id="b9cde-114">Consider a GridView control that includes a TemplateField with a Label Web control with an `ID` value of `ProductName`.</span></span> <span data-ttu-id="b9cde-115">GridView 繫結至其資料來源，在執行階段，此標籤會針對每個 GridView 資料列重複一次。</span><span class="sxs-lookup"><span data-stu-id="b9cde-115">When the GridView is bound to its data source at runtime, this Label is repeated once for every GridView row.</span></span> <span data-ttu-id="b9cde-116">每個呈現標籤需求唯一`id`值。</span><span class="sxs-lookup"><span data-stu-id="b9cde-116">Each rendered Label needs a unique `id` value.</span></span>

<span data-ttu-id="b9cde-117">若要處理這種情況下，ASP.NET 會允許特定的控制項，以表示為命名容器。</span><span class="sxs-lookup"><span data-stu-id="b9cde-117">To handle such scenarios, ASP.NET allows certain controls to be denoted as naming containers.</span></span> <span data-ttu-id="b9cde-118">做為新的命名容器`ID`命名空間。</span><span class="sxs-lookup"><span data-stu-id="b9cde-118">A naming container serves as a new `ID` namespace.</span></span> <span data-ttu-id="b9cde-119">會出現在命名容器中任何伺服器控制項有其呈現`id`值前面加上`ID`命名的容器控制項。</span><span class="sxs-lookup"><span data-stu-id="b9cde-119">Any server controls that appear within the naming container have their rendered `id` value prefixed with the `ID` of the naming container control.</span></span> <span data-ttu-id="b9cde-120">例如，`GridView`和`GridViewRow`類別是這兩個命名的容器。</span><span class="sxs-lookup"><span data-stu-id="b9cde-120">For example, the `GridView` and `GridViewRow` classes are both naming containers.</span></span> <span data-ttu-id="b9cde-121">因此，一個 Label 控制項，定義於與 GridView TemplateField `ID` `ProductName`指定呈現`id`的值`GridViewID_GridViewRowID_ProductName`。</span><span class="sxs-lookup"><span data-stu-id="b9cde-121">Consequently, a Label control defined in a GridView TemplateField with `ID` `ProductName` is given a rendered `id` value of `GridViewID_GridViewRowID_ProductName`.</span></span> <span data-ttu-id="b9cde-122">因為*GridViewRowID*都是唯一的每個 GridView 資料列，產生`id`值是唯一的。</span><span class="sxs-lookup"><span data-stu-id="b9cde-122">Because *GridViewRowID* is unique for each GridView row, the resulting `id` values are unique.</span></span>

<span data-ttu-id="b9cde-127">命名的容器不只變更轉譯`id`屬性值，但也會影響如何控制項可能會從 ASP.NET 頁面的程式碼後置類別以程式設計方式參考。</span><span class="sxs-lookup"><span data-stu-id="b9cde-127">Naming containers not only change the rendered `id` attribute value, but also affect how the control may be programmatically referenced from the ASP.NET page's code-behind class.</span></span> <span data-ttu-id="b9cde-128">`FindControl("controlID")`方法通常用來以程式設計方式參考的 Web 控制項。</span><span class="sxs-lookup"><span data-stu-id="b9cde-128">The `FindControl("controlID")` method is commonly used to programmatically reference a Web control.</span></span> <span data-ttu-id="b9cde-129">不過，`FindControl`無法穿透透過命名容器。</span><span class="sxs-lookup"><span data-stu-id="b9cde-129">However, `FindControl` does not penetrate through naming containers.</span></span> <span data-ttu-id="b9cde-130">因此，您無法直接使用`Page.FindControl`參考 GridView 或其他命名的容器內控制項的方法。</span><span class="sxs-lookup"><span data-stu-id="b9cde-130">Consequently, you cannot directly use the `Page.FindControl` method to reference controls within a GridView or other naming container.</span></span>

<span data-ttu-id="b9cde-131">因為您可能會有猜想，主版頁面和 ContentPlaceHolders 會同時實作為命名容器。</span><span class="sxs-lookup"><span data-stu-id="b9cde-131">As you may have surmised, master pages and ContentPlaceHolders are both implemented as naming containers.</span></span> <span data-ttu-id="b9cde-132">在本教學課程中我們將探討如何主版頁面影響 HTML 項目`id`值，以及如何以程式設計方式參考 [內容] 頁面，使用的 Web 控制項`FindControl`。</span><span class="sxs-lookup"><span data-stu-id="b9cde-132">In this tutorial we examine how master pages affect HTML element `id` values and ways to programmatically reference Web controls within a content page using `FindControl`.</span></span>
